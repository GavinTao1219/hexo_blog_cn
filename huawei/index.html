华为工作经历
软件开发 (物联网方面)
2018年11月 - 2019年6月
中国 - 深圳

华为技术有限公司是一家中国的ICT（信息与通信）基础设施和智能终端提供商，我曾在华为的物联网部门下面的车联网项目群从事软件开发工作。主要负责华为车联网云平台的后端开发。

我使用了Java及多种开源组件，负责华为车联网平台与南向车载设备之间交互业务的开发，实现了基于HTTP协议的Ecall消息上报、基于MQTT协议的车辆控制命令下发及响应上报、采集，保证了代码的单元测试覆盖率与功能稳定，并成功在欧洲上现网使用。

一、主要工作内容

1. 基于HTTP的eCall消息上报

eCall消息是欧盟倡议的，旨在为欧盟内任何地方发生碰撞的驾车者提供快速援助。

  • 接收车载设备发送的HTTP POST请求，并对POST请求体中的16进制码流进行解码，实现了将码流解码为包含具体车辆求救信息的JSON结构体。
  • 作为Kafka生产者，将解码的消息推送到消息队列，供华为车联网平台的其他微服务组件消费后写入数据库、打点记录，并进行具体的业务处理。Kafka消息队列的运用保证了高并发时的稳定性。
  • 使用xml文件与正则表达式，适配了动态url的POST请求，实现了从url中动态读取所需参数。
  • 符合OSGi规范，实现了此功能注册为OSGi的声明式服务，可作为插件（Karaf中的bundle）进行热插拔式的安装、卸载。

2. 基于MQTT的车辆控制命令下发

  • 构建华为车联网平台与车载设备之间的MQTT协议通道，实现了华为车联网平台与车辆设备之间基于MQTT协议的双向通信，保证了糟糕网络环境下服务的稳定性。
  • 实现了短信唤醒车辆设备的特性，并通过订阅MQTT消息的topic，接收车辆被唤醒后主动上报的MQTT消息，并作为Kafka生产者推送给其他微服务组件。
  • 向其他微服务组件提供远程控制命令下发的API，调用该API即可通过MQTT通道向车辆设备发送多种不同的远程控制命令，实现了包括远程开关门锁、鸣笛、开关大灯、开关或调整空调、查询车辆具体信息等。
  • 实现了MQTT topic的订阅，监听车辆设备上报的控制命令响应，并作为Kafka生产者，将响应报文推送给其他微服务组件消费。
  • 实现了多节点的MQTT命令下发与响应，通过在MongoDB中注册路由信息，保证了同一命令与响应均由同一节点进行业务处理。
  • 符合OSGi规范，实现了此功能注册为OSGi的声明式服务，可作为插件（Karaf中的bundle）进行热插拔式的安装、卸载。

二、相关技术栈

1. OSGi + Apache Karaf

以上所有功能开发均符合OSGi架构，各业务模块都作为声明式服务注册发布，并由Maven打包为单独jar包，可热插拔式地单独部署到Apache Karaf中，实现随时安装、运行、停止、卸载等。

2. Apache Kafka

华为车联网平台使用Kafka这种发布/订阅式的消息队列系统来处理各个微服务组件之间的通信问题。以上所有功能开发均使用了Apache Kafka，为其他微服务组件生产消息。

3. Kubernetes

华为车联网平台使用了Kubernetes维护多个微服务，每个微服务都有多个节点的同时运行。日常开发工作中，我熟练掌握了Kubernetes的使用及常用命令，如节点的查找、重建等操作。

4. MQTT协议

MQTT协议是一种广泛应用于物联网领域的基于TCP/IP的发布/订阅式通讯协议，上文中车辆控制命令的下发与响应就是通过不同的MQTT topic实现的不同业务功能的消息通信。
